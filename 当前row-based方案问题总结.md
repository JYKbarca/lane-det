# 当前 row-based 方案问题总结

## 一、当前方案的阶段性结论

当前方案已经从早期的连续回归版本，演进为统一的网格分类版本，主线逻辑为：

- 固定最多 `K=5` 条车道槽位
- 对每条车道在固定 `Y=56` 个 row 上做位置分类
- 图像宽度划分为 `W=100` 个网格
- 每个 row 预测 `W+1` 类
  - 前 `W` 类表示横向位置网格
  - 第 `W+1` 类表示该 row 无车道线
- 通过期望解码恢复连续坐标

从完整训练日志 `train2.log` 和 `vis4` 可视化结果看，这条技术路径是有效的，比早期连续回归版本更有希望，而且在当前 20 个 epoch 内仍在继续收敛；但它还没有达到稳定、可用的状态。

## 二、目前已经确认的改进

### 1. 表示方式比连续回归更合理

新方案将原来的：

- `exist`
- `valid`
- `x_coords`

改成：

- `exist`
- `grid_logits`

这样做的收益是：

- 避免了 `valid` 和 `coord` 两个分支语义不一致
- 分类任务通常比直接回归单个连续坐标更稳定
- 更容易抑制早期出现的“乱飞”“极端折返”“整条假线”问题

### 2. 排序逻辑比旧版本更稳定

`RowTargetBuilder` 已经从“最后 3 个有效点平均 x 排序”改成“所有有效点平均 x 排序”。

这缓解了以下问题：

- 遮挡导致的排序抖动
- 短线样本排序不稳定
- 同一条车道在不同图片中落入不同 slot

### 3. 完整训练过程显示方案在持续收敛

完整训练日志显示，模型并不是在早期短暂提升后就失效，而是在 20 个 epoch 内持续刷新 best。

- 早期较好阶段出现在 `epoch 4`
  - `Accuracy = 0.578830`
  - `FP = 0.819521`
  - `FN = 0.812477`
- 当前完整训练的 best 出现在 `epoch 20`
  - `Accuracy = 0.756733`
  - `FP = 0.578499`
  - `FN = 0.577670`
- `epoch 19` 的 `FP/FN` 略优于 `epoch 20`
  - `Accuracy = 0.752078`
  - `FP = 0.571133`
  - `FN = 0.570304`

这说明新方案不仅不是无效尝试，而且在当前训练配置下仍有继续收敛空间；当前问题已经从“训不起来”转向“几何质量和困难场景鲁棒性还不够”。

## 三、当前仍然存在的主要问题

## 1. 训练整体在持续收敛，但还没有达到成熟稳定状态

完整日志表明，验证集指标并不是在 `epoch 4` 后一路变差，而是在中间存在局部波动的同时继续向上提升，直到 `epoch 20` 仍在刷新 `Accuracy` best。

这说明：

- 当前方案不是“前几轮有效、后面失效”
- 继续训练和学习率衰减在当前配置下仍然有效
- 但还不能把“指标继续上涨”等同于“几何质量已经解决”

因此当前更准确的判断不是“训练后期回落”，而是“训练在收敛，但模型还没有进入真正成熟、可稳定泛化的阶段”。

## 2. 几何贴线精度仍不足，仍存在系统性横向偏移

从 `vis4` 的可视化结果看，当前预测已经能较好对齐主结构，尤其在直道和部分缓弯场景中，左右边界线大体能够跟住 GT；但仍然存在明显几何误差：

- 红线经常整体偏到 GT 的内侧或外侧
- 中间车道比边界车道更容易偏移
- 局部位置虽然接近真实车道，但还不够稳定贴合

这说明：

- 模型已经不再是“乱画”
- 但仍没有稳定学会高精度地贴住真实车道几何
- 当前误差更像“结构对了，但位置还差一段”

## 3. 局部连续性和遮挡处鲁棒性不足

`vis4` 中可以看到，特别是在中间车道和车辆遮挡附近，预测仍容易出现以下现象：

- 局部短碎段
- 轻微锯齿、波浪形或折返感
- 前后段衔接不自然，连续性不足

这说明：

- 当前模型已经能够输出整条 lane 的大致趋势
- 但在局部 row-to-row 一致性上还不够稳
- 几何解码后的连续曲线质量仍是主要短板

## 4. 弯道与特殊场景下的稳定性仍不够

排序逻辑改成“所有有效点平均 x”之后，slot 抖动问题已经明显缓解，但在强弯道和特殊场景下仍有风险。

潜在问题主要体现在：

- 强弯道场景下，全局平均 `x` 可能不能准确代表近场左右关系
- 弯道中不同车道的整体均值可能更接近，导致 slot 语义变弱
- 可视化中弯道样本仍能看到更明显的偏移、断裂或局部段落错误

所以当前状态应理解为：

- slot 对齐问题已不再是唯一主矛盾
- 但它仍然会在困难场景中放大几何误差

## 5. 假阳性和漏检虽然明显下降，但仍不可忽视

相比早期接近 `1.0` 的极端状态，当前 `FP/FN` 已经大幅下降；但即使在 `epoch 19~20`，它们仍然在 `0.57` 左右，说明误检和漏检还没有降到足够低的水平。

这说明：

- 预测结果中仍然存在错误车道
- 同时也仍会漏掉部分真实车道
- 当前模型还不能稳定给出“数量合适、位置正确、几何可信”的车道输出

## 6. 当前 best checkpoint 选择标准不够全面

目前 `best.pth` 仍然只按 `Accuracy` 选择。

这存在问题：

- `Accuracy` 高不一定代表 `FP/FN` 更平衡
- 当前阶段视觉效果和评估数值并不完全同步
- 单独按 `Accuracy` 选 best，可能错过更平衡的 checkpoint

因此当前 checkpoint 选择标准存在明显优化空间。

## 四、已经排除或弱化的问题

## 1. 不是 Backbone/FPN 失效

当前 `ResNet18 + FPN` 明显在正常起作用：

- loss 在正常下降
- 模型能输出接近车道的结构
- 指标在完整训练过程中持续改善

所以当前的主问题不在 backbone，而在：

- 输出表示
- slot 对齐
- 几何监督与解码方式

## 2. 不是简单阈值问题

之前已经做过多组阈值扫描，视觉主形态没有本质变化。

说明：

- 不是单纯 `exist_score_thr` 太低
- 也不是后处理阈值过松导致的假线泛滥

问题更深层，主要在模型本身输出的 lane 语义和几何质量上。

## 3. 不是继续调平滑权重就能解决的问题

之前尝试过更强的平滑损失和 query 正则，结果表明：

- 线条会更顺
- 但会更模板化
- 同时更容易整体偏移

说明单纯加强几何约束不是主线解决方案。

## 五、当前方案相比旧方案的真实位置

当前 grid-based 方案相较旧的连续回归版，处于如下状态：

- 在技术方向上更合理
- 在实验结果上已经形成明显领先
- 在训练收敛趋势上已经得到验证
- 但在几何精度、局部连续性和困难场景稳定性上仍未成熟

也就是说：

它已经不是“试错失败的分支”，而是“值得继续作为主线推进的方案”。

## 六、当前最核心的问题归纳

如果只保留一句总结，当前问题可以概括为：

> 新方案已经学会了更稳定地生成车道结构，并且在当前训练配置下持续收敛，但仍然没有稳定学会“在遮挡、弯道和中间车道上，将每条车道准确、平滑、连续地贴合到真实车道线位置”。

进一步拆开就是三点：

1. 主体结构已经对上，但仍有系统性偏移
2. 局部连续性不足，遮挡和中间车道更容易出问题
3. slot 对齐有所改善，但弯道和特殊场景下的监督语义仍未完全稳定

## 七、后续工作建议

后续不建议再回到旧的连续回归框架，而应继续沿当前 grid-based 主线推进。

但接下来的推进方式需要更明确地按主线步骤展开，避免“哪里有问题就加一点兜底逻辑”的碎片化优化。

### 0. 实现原则

后续所有优化都应遵循下面的原则：

- 优先修改真正决定检测效果的主干环节：表示、监督、解码、训练数据
- 优先做能提升“贴线精度、连续性、困难场景鲁棒性”的主线改动
- 尽量避免堆积大量兜底规则、特判后处理和临时修补逻辑
- 不要为了让个别样本看起来更顺而明显牺牲代码可读性、结构清晰度和整体可维护性
- 如果某个改动主要收益来自“补洞式修线”而不是“让模型本身学得更准”，应降低优先级

### 1. 第一步：优先修正 row head，使其保留横向空间信息（代码改造已完成，效果待重新训练验证）

当前最优先的问题，不是继续调阈值或加后处理，而是 head 本身对横向位置信息利用不够充分。

当前已完成的改动：

1. 不再过早把宽度维压成 `1`
2. 先保留 `row-grid` 横向结构，再进行位置分类
3. 将每个 lane slot 的位置打分建立在显式的 row-grid 特征上，而不是仅依赖全局压缩后的 row 特征
4. 将“no-lane”打分从位置网格打分中分离出来，保持语义更清晰

本次完成的是结构层面的第一步改造，当前只能确认“表示方式更合理”；是否能够稳定带来指标和可视化收益，还需要重新训练后再验证。

这一改动直接对应的核心意图是：

- 不让模型在预测 `x` 网格之前先丢掉横向空间信息
- 让中间车道和相邻车道之间的横向差异能够更直接地被建模
- 从表示层面减轻“模板线”和整体横向偏移问题

后续重新训练时，应重点验证这一步是否改善：

- 整体横向偏移
- 中间车道定位质量
- 弯道场景下的几何贴合能力

### 2. 第二步：对齐监督与解码，重点提升贴线精度

当前训练时是离散分类监督，推理时再通过期望解码恢复连续坐标，这两者之间还不够对齐。

需要做的事情：

1. 在现有 grid 分类之外，增加直接约束连续坐标的损失
2. 将“单个 bin 的硬标签”改成更平滑、更接近期望解码语义的监督方式
3. 让训练目标直接鼓励 `expected_x` 更贴近真实车道位置，而不是只要求分类命中某一个网格

这一阶段的目标是：

- 减少整条线系统性偏内/偏外
- 提升亚网格级别的位置精度
- 让可视化结果从“接近 GT”进一步变成“贴住 GT”

### 3. 第三步：拆清 row 是否存在车道 与 row 的横向位置 两种语义

当前 `W+1` 分类把“位置”和“该 row 无车道”混在一起，这不利于局部连续性和遮挡下的稳定预测。

需要做的事情：

1. 将 row 有效性判断与位置分类分开建模
2. 只在有效 row 上学习横向位置
3. 让解码先判断该 row 是否应该有点，再恢复坐标

这一阶段的目标是：

- 减少局部短碎段
- 减少遮挡附近的断裂和误连
- 提高 row-to-row 连续性

### 4. 第四步：以轻量方式加强局部连续性，而不是堆复杂兜底逻辑

当前局部锯齿、轻微折返和前后段衔接不自然，说明连续性约束还不够，但这不意味着应该堆大量后处理规则。

需要做的事情：

1. 优先在训练目标里增强局部连续性约束
2. 只保留少量必要、简单、可解释的解码约束
3. 避免引入大量按场景特判的修线规则

建议的方向是：

- 优先通过监督和结构设计解决
- 后处理只保留最基础、最稳定的连续性约束
- 不把后处理做成“第二个模型”

### 5. 第五步：继续优化弯道和特殊场景下的 slot 排序稳定性

“所有有效点平均 x 排序”已经比旧版本更稳，但它仍然更像过渡方案，而不是最终方案。

需要做的事情：

1. 重点分析弯道场景下 slot 是否仍存在错位
2. 让排序依据更贴近近场真实左右关系，而不是只依赖全局均值
3. 避免因为排序不稳放大几何偏移

这一阶段的目标是：

- 减少弯道样本中的 slot 语义漂移
- 减少中间车道在特殊场景中的错分和偏移

### 6. 第六步：补强训练数据和增强方式，但只做真正服务主问题的增强

当前数据增强主要是翻转和颜色扰动，对弯道、遮挡、局部断裂这类问题帮助有限。

需要做的事情：

1. 增加更贴近真实误差来源的增强方式
2. 重点覆盖遮挡、透视变化、局部扰动等困难因素
3. 不为了“看起来增强很多”而加入大量收益不明确的复杂增强

这一阶段的目标是：

- 提升困难场景鲁棒性
- 提升泛化质量
- 让改进直接服务当前暴露出来的误差模式

### 7. 第七步：重做 checkpoint 选择与误差分析流程

当前只按 `Accuracy` 选 best 不够全面，也会影响对模型真实水平的判断。

需要做的事情：

1. 联合考虑 `Accuracy`、`FP`、`FN` 和可视化质量
2. 固定一套统一的 best 模型筛选标准
3. 每轮重点看中间车道、遮挡区域、弯道样本的误差模式

这一阶段的目标是：

- 让“最佳模型”定义更可靠
- 避免被单一指标误导
- 让后续优化建立在更可信的分析基础上

### 8. 第八步：在主结构改完后，再确认训练上限

完整日志显示当前配置在 `epoch 20` 仍未明显平台化，但在主结构问题未解决前，单纯继续拉长训练并不是最高优先级。

需要做的事情：

1. 先完成前面的关键结构改动
2. 再重新观察学习率调度和训练轮数的上限
3. 在新结构上确认是否还需要更长训练

这一阶段的目标是：

- 避免在次优结构上反复堆训练时间
- 把训练资源优先用在正确的模型形态上

## 八、当前结论

当前 row-based 技术路径已经完成了一次重要升级：

- 从连续回归转向统一网格分类
- 从分裂的 `valid + coord` 表示转向统一的结构化位置分类

它已经证明自己比旧版本更有潜力，而且当前训练过程也显示出明确的持续收敛趋势；但它还没有完成最终收敛。

现阶段最准确的判断是：

> 这条路径值得继续，而且当前重点已经不再是“能不能训起来”，而是“如何把已经学到的结构能力进一步变成更稳定的贴线精度、局部连续性和困难场景泛化能力”。
